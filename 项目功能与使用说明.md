# POE2 市集汉化增强版 项目功能与使用说明

## 1. 这个项目是做什么的

这是一个基于 `Chrome Extension Manifest V3` 的浏览器扩展，主要用于给 `Path of Exile 2` 相关站点提供汉化和部分增强功能。

从代码来看，它当前的核心目标是：

- 汉化 `Path of Exile 2 Trade / Trade2` 市集页面
- 汉化 `poe.ninja` 页面内容
- 为 `poe.ninja` 里的物品增加一键跳转市集搜索能力
- 支持简体中文、繁体中文、英文原文切换
- 对部分第三方站点做额外翻译支持或预留接入

## 2. 需要什么运行环境

这个项目不是 Node.js、Python 或 Java 项目，不需要安装依赖，也不需要执行 `npm install`、`pnpm install`、`yarn`、`python` 之类的命令。

运行环境很简单：

- 一个支持 Chrome 扩展的 Chromium 内核浏览器
- 推荐：`Google Chrome`、`Microsoft Edge`、`360 极速浏览器（Chromium 内核）`
- 需要能访问扩展目标网站
- 需要允许浏览器加载“已解压的扩展程序”

补充说明：

- 扩展版本：`2026.5.22`
- 扩展类型：`Manifest V3`
- 本项目没有后端服务
- 本项目没有本地启动命令
- 本项目的翻译数据直接放在项目目录的 `data/` 下

## 3. 怎么使用

### 3.1 安装方式

1. 打开浏览器扩展管理页
2. 开启“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择当前项目根目录：

`C:\Users\wangh\Downloads\googlepoe2\【谷歌插件】POE2 市集汉化增强版`

5. 安装完成后，浏览器工具栏会出现扩展图标

### 3.2 基础使用

1. 点击浏览器工具栏中的扩展图标
2. 在弹出窗口中选择语言：
   - `简体中文`
   - `繁體中文`
   - `English (原文)`
3. 扩展会自动刷新当前标签页，让语言设置生效

### 3.3 详细设置

点击扩展弹窗中的 `打开详细设置`，可以进入设置页。

设置页支持：

- 全局语言切换
- 按站点单独设置语言
- 按站点启用/关闭功能
- 设置 `poe.ninja` 跳转市集时使用的交易服务器

### 3.4 修改后如何生效

通常会自动生效；如果当前页面没有立即变化，可以手动刷新一次页面。

## 4. 项目里有哪些主要功能

## 4.1 Trade / Trade2 市集汉化

作用：

- 汉化国际服 Trade / Trade2 页面中的界面文本
- 汉化搜索项、词缀、部分物品名、结果列表字段
- 在英文原文旁显示中文，便于对照

适用网址：

- `https://www.pathofexile.com/trade/*`
- `https://www.pathofexile.com/trade2/*`

怎么用：

1. 打开 POE2 官方市集页面
2. 确保扩展中 `Trade` 功能已启用
3. 选择需要的语言
4. 刷新页面
5. 页面中的筛选、结果和部分词缀会自动汉化

说明：

- 切到 `English` 时，扩展会清理本地翻译缓存并回到英文原文
- 翻译数据来自项目内置的 `data/zh_cn/trade/` 和 `data/zh_tw/trade/`

## 4.2 poe.ninja 汉化

作用：

- 汉化 `poe.ninja` 的界面文案
- 汉化物品、天赋、部分技能、词缀等内容
- 支持大量 JSON 词典和模板化词缀翻译

适用网址：

- `https://poe.ninja/*`

怎么用：

1. 打开 `poe.ninja`
2. 确保扩展中 `Ninja` 功能已启用
3. 在扩展弹窗或设置页里选择语言
4. 刷新页面
5. 页面会自动进行汉化

说明：

- `data/zh_cn/ninja/` 和 `data/zh_tw/ninja/` 是 Ninja 的核心翻译库
- 其中分为 `simple/` 简单词典 和 `modifiers/` 复杂词缀翻译

## 4.3 poe.ninja 一键跳转市集搜索

这是这个项目里最实用的增强功能之一。

作用：

- 在 `poe.ninja` 物品详情上提取物品信息
- 选择你关心的词条后，一键跳转到官方市集搜索
- 可按国际服 / 台服 / 国服构造对应搜索链接

怎么用：

1. 打开 `poe.ninja`
2. 把鼠标移动到物品上，先让物品详情浮窗出现
3. 按一下键盘 `Alt` 键，固定当前物品详情浮窗
4. 在固定后的浮窗里选择你要带入搜索的词条
5. 点击浮窗工具栏中的 `市集` 按钮
6. 扩展会自动新开一个对应的 Trade 搜索页面

补充功能：

- 可以选择是否把 `Item Level`、`Quality` 一起带入搜索
- 可以点击 `复制` 按钮，把当前选中的物品参数复制到剪贴板

使用建议：

- 如果搜索失败，先确认你是否已经登录对应的官方交易站
- 如果提示请求过于频繁，稍等片刻再试
- 如果切换到国服搜索，扩展会优先把物品名转换成中文后再发起搜索

### 4.3.1 设置 Ninja 使用哪个交易服务器

在详细设置页中可以设置：

- `国际服`
- `台服`
- `国服`

这个设置会影响：

- 一键跳转市集时使用哪个站点
- 联盟名称如何映射
- 搜索时使用英文还是中文物品名

## 4.4 Maxroll 汉化

作用：

- 对 `maxroll.gg/poe2` 页面中的部分文本进行翻译
- 包括固定文案、技能名称、传奇名、部分模板化属性

适用网址：

- `https://maxroll.gg/poe2/*`

怎么用：

1. 进入详细设置页
2. 打开 `Maxroll` 开关
3. 选择 `Maxroll` 使用的语言
4. 刷新 `maxroll.gg/poe2` 页面

说明：

- 该功能依赖 `data/maxroll/` 下的一组映射表
- 从代码看，`Maxroll` 有实际翻译逻辑，但默认是关闭的

## 4.5 FilterBlade / Mobalytics 支持情况

这两个站点在设置页中都有开关：

- `FilterBlade`
- `Mobalytics`

从当前代码实际情况看：

- 已经有内容脚本注入
- 会读取站点开关和语言设置
- 会尝试加载基础词典
- 但目前没有看到完整的页面翻译替换逻辑

也就是说：

- 这两个功能更像“预留接入”或“半成品”
- 设置里可以开启，但未必会有明显可见的完整汉化效果

如果你要实际使用，建议先把它理解为：

- 已挂载入口
- 可能还在开发中
- 当前主力功能不是它们

## 4.6 Poe2db 支持情况

扩展会在下面的网址注入脚本：

- `https://poe2db.tw/us/*`

但从当前代码看：

- 脚本只读取语言设置
- 没有看到实际执行文本替换的逻辑

所以目前更准确的结论是：

- `Poe2db` 处于预留状态
- 当前代码里没有形成完整可见功能

## 5. 设置项怎么用

## 5.1 全局语言

用途：

- 控制整个扩展的默认语言

可选值：

- `zh_cn`：简体中文
- `zh_tw`：繁体中文
- `en`：英文原文

建议：

- 想彻底关闭汉化时，直接选 `English`

## 5.2 站点独立设置

用途：

- 给不同站点分别设置语言和开关

目前设置页中可配置的站点有：

- `Trade`
- `Ninja`
- `FilterBlade`
- `Mobalytics`
- `Maxroll`

怎么用：

1. 打开详细设置页
2. 找到对应站点
3. 选择语言
4. 打开或关闭右侧开关
5. 刷新目标站点页面

## 5.3 重置默认设置

点击设置页底部的 `重置默认` 后，会恢复为以下默认值：

- 全局语言：`zh_cn`
- `Trade`：启用
- `Ninja`：启用
- `FilterBlade`：关闭
- `Mobalytics`：关闭
- `Maxroll`：关闭
- `Ninja 交易服务器`：`国际服`

## 6. 项目目录里各部分是做什么的

### 根目录

- `manifest.json`：扩展配置文件
- `popup.html` / `popup.js`：扩展弹窗
- `options.html` / `options.js`：详细设置页
- `README.md`：原项目说明

### js/

- `js/background.js`：后台服务，负责加载翻译数据、缓存、发起市集搜索
- `js/trade/`：Trade 页面相关逻辑
- `js/ninja/`：poe.ninja 汉化和一键搜索逻辑
- `js/other/`：其他网站脚本
- `js/config/`：语言和数据库配置

### data/

- `data/zh_cn/`：简体中文翻译数据
- `data/zh_tw/`：繁体中文翻译数据
- `data/common/`：公共数据
- `data/maxroll/`：Maxroll 专用映射数据
- `data/fallback/`：兜底数据

## 7. 这个项目怎么维护

如果你只是“使用扩展”，不需要改代码。

如果你要“维护翻译内容”，通常改这些位置：

- Trade 文本：`data/zh_cn/trade/`、`data/zh_tw/trade/`
- Ninja 文本：`data/zh_cn/ninja/`、`data/zh_tw/ninja/`
- Maxroll 文本：`data/maxroll/`

修改步骤：

1. 编辑对应的 `json` 或 `js` 映射文件
2. 保持文件编码为 `UTF-8`
3. 保持 JSON 格式正确
4. 回到浏览器扩展管理页，点击刷新扩展
5. 重新打开目标网页验证效果

## 8. 实际使用建议

如果你是普通用户，推荐这样用：

1. 先只启用 `Trade` 和 `Ninja`
2. 全局语言选 `简体中文` 或 `繁体中文`
3. 需要查装备时，用 `poe.ninja + Alt 固定物品详情 + 市集按钮` 跳转搜索
4. 想看原文时，再切到 `English`

如果你是维护者，推荐优先关注：

- `Trade` 汉化是否完整
- `Ninja` 词典是否缺词
- `Maxroll` 翻译映射是否还需要补充
- `FilterBlade`、`Mobalytics`、`Poe2db` 是否要继续补齐真正的翻译逻辑

## 9. 当前项目结论

从代码现状看，这个项目最成熟、最值得实际使用的功能是：

- `POE2 Trade / Trade2 汉化`
- `poe.ninja 汉化`
- `poe.ninja 一键跳转市集搜索`
- `Maxroll 基础翻译支持`

而下面这些功能目前更像预留或未完成：

- `FilterBlade`
- `Mobalytics`
- `Poe2db`

如果你的目标是“直接使用”，这个扩展已经够用；
如果你的目标是“继续开发”，最适合优先补完的就是那几个预留站点的实际翻译逻辑。
